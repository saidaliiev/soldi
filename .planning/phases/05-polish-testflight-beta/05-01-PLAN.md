<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| device user → app open | An unauthorized holder of an unlocked phone could open the financial app |
| app → iOS share-sheet / filesystem cache | Exported financial CSV crosses into a shared/temp path and other apps |
| app state → secure-store | Biometric-enabled flag must not spill to AsyncStorage |
| iOS app switcher | Backgrounded app snapshot may show financial data on the multitasking screen |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Elevation / Spoofing | biometric gate `app/_layout.tsx` | mitigate | Render gate returns null until `biometricPassed`; never render partial UI on failed auth; resume listener re-locks after > 5 min. ASVS L1 auth-on-open. |
| T-05-02 | Bypass | failed-auth retry | mitigate | `disableDeviceFallback:false` → OS passcode fallback; no app-level retry cap (rely on iOS biometric lockout) — D-01. |
| T-05-03 | Information Disclosure | `exportRepo.ts` CSV in `cacheDirectory` | mitigate | Files written to app-sandboxed `FileSystem.cacheDirectory` (not world-readable); delivered only via user-initiated `expo-sharing` share-sheet; OS reclaims cache. No background/auto export. |
| T-05-04 | Information Disclosure | onboarding store persistence | mitigate | `biometricEnabled` persisted only inside the existing `'soldi-onboarding'` secure-store blob; grep-gate asserts zero AsyncStorage import in store + toggle. |
| T-05-05 | Information Disclosure | iOS app switcher snapshot | accept | v1 portfolio scope; no privacy-overlay/blur on background documented as accepted risk (low value target, local-only data). Revisit if real bank data added. |
| T-05-06 | Information Disclosure | export error logging | mitigate | Catch logs `err.name` only, never row data/amounts (CLAUDE.md); `__DEV__`-gated console only. |
| T-05-SC | Tampering | npm installs | mitigate | No new packages introduced (expo-local-authentication / expo-sharing / expo-file-system already deps per app.json + PATTERNS). No legitimacy checkpoint needed. |
</threat_model>
