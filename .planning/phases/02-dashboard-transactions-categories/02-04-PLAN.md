## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Category name input → SQL INSERT/UPDATE | User-typed text; rendered later as Text (not HTML) but sanitization still warranted |
| Drag-drop merge → bulk transactions UPDATE | User gesture triggers a multi-row write; race-condition surface |
| Default categories → delete UI | Default rows must not be deletable |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-01 | Tampering | categoryMutations.createCategory / renameCategory | mitigate | validateCategoryName rejects empty, length > 40, regex-banned characters `<>"'`;\\`, and duplicate (case-insensitive); SQL uses parameterized executeSync — never string concatenation |
| T-02-04-02 | Tampering | bulkReassignTransactionsCategory + deleteCategoryRow | mitigate | Wrapped in BEGIN/COMMIT executeSync transaction (splitStatements); deleteCategoryRow SQL has `AND is_default = 0` guard so even a malformed call cannot delete a default category |
| T-02-04-03 | Denial of Service | mergeCategories on huge category | accept | Worst case ~5000 transactions per category (synthetic-data ceiling); single UPDATE statement is bounded; executeSync is synchronous, no UI thread blocking beyond ~50ms even on SE 2020 |
| T-02-04-04 | Race condition | drag-drop merge with stale memory state | mitigate | Before issuing mergeCategories, re-resolve both ids via categoriesRepo.getCategoryById; if either returns null (category deleted in background), surface "Couldn't merge — try again" toast and abort. Document the race in code comment per planning_context security note. |
| T-02-04-05 | Repudiation | merge has no undo | accept | UI-SPEC §Destructive Confirmations copy explicitly says "This cannot be undone." User opt-in via confirm modal; matches D-19 design intent |
| T-02-04-06 | Information disclosure | Category name in console.log | mitigate | No console.log of name in production paths; grep gate in verify task scans for `console.log\\(.*name` |
| T-02-04-07 | Spoofing | Slug collision on rapid creates | mitigate | createCategory wrapped in try/catch; on `UNIQUE constraint failed: categories.slug`, append `-{rand}` and retry once; if still fails throw and surface error_save toast |
