## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User search input → SQL LIKE | Free-form text bound as parameter to LOWER(merchant_name) LIKE ? |
| Amount range numeric inputs → SQL BETWEEN | User-typed decimal strings parsed to cents |
| Date range pickers → SQL BETWEEN | ISO date strings from native picker |
| Filter state → secureStorage persistence | Zustand-persisted; secure-store rather than AsyncStorage |
| Transaction detail edit → UPDATE | merchant_name, amount_cents, occurred_at, category_id user-editable |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Tampering | filterCompose.buildFilterSql | mitigate | All user values bound via positional `?` parameters; no template-string interpolation of user input; LIKE pattern wraps user text with `%` ON THE BIND SIDE (`%${val}%` builds the bound string, not the SQL). Whitelist of axes — unknown keys ignored |
| T-02-03-02 | Tampering | transactionsRepo.updateTransaction | mitigate | Whitelist of four columns (merchant_name, amount_cents, occurred_at, category_id); unknown keys throw before SQL is built; SET clause built programmatically from whitelist, not from user-supplied keys |
