## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-01 | Tampering | SQL injection via merchant_name | mitigate | All INSERTs use parameterized queries via op-sqlite execute(sql, params) |
| T-01-03-02 | Denial of Service | huge generator output exhausts memory | mitigate | Inputs are bounded (days ≤ 90, maxPerDay ≤ 6); ceiling is ~540 + salary rows; well under any RN heap concern |
| T-01-03-03 | Information Disclosure | logging seeded payload | mitigate | No console.log of row contents; only counts in UI |
| T-01-03-04 | Tampering | re-running generator duplicates data | mitigate | UNIQUE(source, external_id) on transactions + INSERT OR IGNORE; deterministic external_id |
| T-01-03-05 | Information Disclosure | currency confusion (UAH vs EUR row) | mitigate | currency stored per-row; UI formats per row in Phase 2 (Phase 1 dashboard uses EUR rollup — accepted simplification, recorded in SKELETON Out-of-Scope) |
</threat_model>
