# Phase 3 — UI Design Contract
## AI Categorization + Chat

> Visual and interaction contract for Phase 3. Generated by gsd-ui-researcher.
> Consumed by gsd-planner (task specs), gsd-executor (implementation source of truth), gsd-ui-auditor (retroactive audit).
> Inherits all token decisions from Phase 2 (`02-UI-SPEC.md`). This document specifies ONLY the new surfaces introduced in Phase 3.
> Do NOT re-design Phase 2 surfaces. Where Phase 3 reuses a Phase 2 component, this spec notes the reuse and any minor extension.

## Scope (UI-only)

Phase 3 ships three plans. Only **03-03** has primary new UI:

| Plan | UI Surface | Spec Section |
|------|-----------|--------------|
| 03-01 `ai-categorize` | None (background Edge Function). One existing-row extension: AI-categorized indicator dot. | §AI Indicator Dot |

**Surfaces explicitly NOT redesigned:**
- Phase 2 `RecategorizeBottomSheet` — reused exactly as specified in `02-UI-SPEC.md`. Propagation logic happens silently after the sheet closes; user feedback is the new PropagationToast.

- Tab bar — no new tab. Chat is launched via a floating action button on the Dashboard tab (see §ChatLaunchFAB), not a tab.

## Typography

All presets from `TYPE` in `apps/mobile/src/design/typography.ts`. Use preset name, never inline values.

### Type Scale for Phase 3

Phase 3 uses **3 active text sizes** in the phase type scale: **28pt, 16pt, 14pt**.

The 3-typeface system continues per CLAUDE.md (Oswald display / EB Garamond editorial / Manrope UI). Each typeface stays within its 2-weight budget. Phase 3 leans **harder on EB Garamond** than Phase 2 because chat is the editorial heart of the product — bubbles use editorial body, KPIs in answers use editorial lead, and the typing indicator falls under the editorial mood. UI primitives (input, buttons, chips, toast) stay Manrope.

`TYPE.displayXL` (64pt) and `TYPE.displayL` (40pt) are **excluded from Phase 3** — chat does not need hero-scale display. The largest type in chat is `TYPE.displayM` (28pt) used once on the sheet header and once per assistant-answer KPI number.

`TYPE.uiMeta` (12pt) is **excluded from Phase 3** for the same legibility reason as Phase 2. Timestamps fall back to `TYPE.uiLabel` (14pt).

| Role | Preset | Font | Size/Weight/LH | Used On |
|------|--------|------|----------------|---------|
| Sheet header title | `TYPE.displayM` | Oswald Medium | 28pt / 500 / 34pt | "Ask SOLDI" header on ChatBottomSheet |
| Assistant answer KPI number | `TYPE.displayM` | Oswald Medium | 28pt / 500 / 34pt + `fontVariant: ['tabular-nums']` | Numeric answer inside assistant bubble (e.g., "€247.50") |

## Animation Contract

All animations use `react-native-reanimated` v4. Worklets only for sheet pan gesture (handled by `@gorhom/bottom-sheet` internals).

| Interaction | Curve | Duration | Notes |
|-------------|-------|----------|-------|
| Chat sheet mount (FAB tap → expand) | `@gorhom/bottom-sheet` internal spring | default gorhom | Snaps to 85% snap point; do not override gorhom |
| Chat sheet drag handle pull-up | gorhom pan-tracker | gesture-driven | Pull down past 30% threshold dismisses |
| Chat sheet dismiss | gorhom spring | default | Backdrop fades to 0 in parallel |
| FAB mount on dashboard | `withDelay(200, withSpring)` scale 0→1 | spring damping 18 | Appears 200ms after dashboard renders (lets content land first) |
| FAB tap | `withTiming` scale 1 → 0.94 → 1 | 120ms total | Visual confirmation of press |
| Empty-state entrance (sheet open, no messages yet) | `withDelay(150, withTiming)` opacity 0→1 + translateY 8→0 | 300ms `Easing.out(Easing.cubic)` | Same pattern as Phase 2 empty states |
| Prompt chip tap | `withTiming` scale 1 → 0.96 → 1 | 100ms | `Haptics.selectionAsync()` fires in parallel |
| Bubble entrance (new message added to list) | `withTiming` opacity 0→1 + translateY 12→0 | 250ms `Easing.out(Easing.cubic)` | Per-bubble entrance; staggered by FlashList index order |
| Typing indicator dots | three `withRepeat(withSequence(withTiming(opacity 0.3→1, 400ms), withTiming(1→0.3, 400ms)))` with 150ms stagger between dots | infinite | Looks like soft breath, not a hard pulse |
| Mini chart entrance | `withDelay(300, withTiming)` Skia path stroke from 0→1 progress | 600ms `Easing.out(Easing.cubic)` | Sparkline draws left-to-right; donut sweeps clockwise |
| Send button tap | `withTiming` scale 1 → 0.92 → 1 | 100ms | `Haptics.impactAsync('light')` fires |
| Send button → loading state | `withTiming` opacity icon → spinner | 150ms cross-fade | Spinner is a Reanimated rotating SVG |
| Error banner appear | `withTiming` opacity 0→1 + translateY -8→0 | 200ms | Tap to dismiss + retry → fade out 150ms |
| PropagationToast appear | `withSpring(translateY: 0)` from 60pt below | spring damping 16 | `Haptics.impactAsync('medium')` fires |
| PropagationToast auto-dismiss | `withTiming(translateY: 60, opacity: 0)` | 250ms after 4000ms idle | Auto-dismiss timer cancelled if user taps |
| PropagationToast undo tap | toast fade out 150ms, transactions DB rollback fires immediately | — | No spinner — DB rollback is synchronous via `executeSync` |
| AI-indicator dot fade-out | `withTiming(opacity 1→0)` | 400ms | Triggers when user has viewed the row (scrolled into viewport for ≥ 500ms) — read-side state managed in Zustand |
| Bubble timestamp reveal (tap bubble) | `withTiming` opacity 0→1 + height 0→18pt | 180ms | Tap same bubble again to hide |

## Component Specifications

### ChatLaunchFAB

**Purpose:** Floating action button on the Dashboard tab that opens ChatBottomSheet.
**Placement:** Bottom-right of `DashboardScreen`, `SPACING.md` (16pt) from right edge, `SPACING.md` (16pt) above the tab bar's top edge. Absolute positioned, does not affect scroll layout. Hides itself when user is at the very top of the scroll (`scrollY > 40pt` → visible; `scrollY ≤ 40pt` → fading hidden) so it doesn't compete with the hero monthly total on first frame.
**Size:** 56×56pt, `RADIUS.pill`.
**Fill:** `GRADIENTS.primary` (`#D9997A` → `#C97B5C`) via `LinearGradient` — spread the tuple per CLAUDE.md (`colors={[...GRADIENTS.primary]}`).
**Icon:** Custom hand-drawn SVG "spark"/"chat-quill" icon in `src/design/icons/system/` — 24×24pt, `COLORS.white`. NOT a generic chat-bubble emoji icon; NOT a sparkle-AI icon. Aesthetic: a quill nib with a small dot (editorial + AI subtle hint without leaning on AI-slop iconography).
**Shadow:** `SHADOWS.card` (subtle, not glowing).
**Pressed state:** `withTiming` scale 0.94 + gradient swaps to `[COLORS.accentSoft, COLORS.accentDeep]`.
**Tap target:** 56×56pt — exceeds 44pt minimum.
**Accessibility:** `accessibilityRole="button"`, `accessibilityLabel="Ask SOLDI"` (`ai.fab_label`), `accessibilityHint="Opens the AI assistant"`.

### ChatBottomSheet

**Purpose:** Primary chat surface. Pull-up sheet covering ~85% of screen.
**Library:** `@gorhom/bottom-sheet`.
**Route:** Not a route — rendered as a portal/overlay child of the `(tabs)` layout. State (open/closed) managed via Zustand `chatStore.isOpen`.
**Snap points:** `['85%']` (single snap, per phase_context).
**Background:** `COLORS.surface` (`#FAF5F0`), `RADIUS.xl` (24pt) top corners only.
**Backdrop:** `COLORS.textPrimary` @ 0.5 opacity — semi-modal (lighter than full-modal 0.9). Tap backdrop dismisses sheet.
**Drag handle:** 4×36pt pill, `COLORS.textMuted` @ 40%, centered, `mt: SPACING.sm` (8pt).
**Header row (56pt height):**
- Left: `"Ask SOLDI"` (`ai.sheet_title`) — `TYPE.displayM` (28pt Oswald) `COLORS.textPrimary`
- Right: Close `×` button (24pt SVG `COLORS.textSecondary`, 44×44pt hit area). `accessibilityLabel="Close"`.
- Bottom border: 1pt `COLORS.textMuted` @ 15%.
**Body area (between header and input row):**
- If `messages.length === 0` → renders `ChatEmptyState` component
- Else → renders `ChatMessageList` (`FlashList`)
**Input area (anchored bottom, above safe area inset):** `ChatInputRow` component, sticky.
**Keyboard handling:** `KeyboardAvoidingView` on iOS with `behavior="padding"`. Sheet content area shrinks; input row stays above keyboard.
**Accessibility:** `accessibilityViewIsModal={true}` on sheet root. Header `accessibilityRole="header"`.

### PromptSuggestionChip

**Purpose:** Tappable suggested-prompt chip in empty state.
**Size:** Height 36pt; horizontal padding `SPACING.md` (16pt) each side; auto-width to fit text plus padding.
**Background:** `COLORS.surface` with 1pt border `COLORS.textMuted` @ 30%, `RADIUS.pill`.
**Label:** `TYPE.uiLabel` (14pt Manrope Medium) `COLORS.textPrimary`. Single-line, no truncation (chips are short by design).
**Pressed state:** Background `COLORS.accent` @ 10%, border `COLORS.accent` @ 60%.
**Tap behavior:** Populates the `ChatInputRow` text field with the chip's prompt text AND submits immediately (no extra tap needed). `Haptics.selectionAsync()` fires. The chip block then disappears (because `messages.length > 0` once the user message is appended).
**Tap target:** 36pt height + 8pt vertical `hitSlop` = 52pt total — exceeds 44pt minimum.
**Accessibility:** `accessibilityRole="button"`, `accessibilityLabel="[Prompt text]"`, `accessibilityHint="Sends this question to the assistant"`.

### ChatMessageList

**Purpose:** Scrollable list of user + assistant messages.
**Library:** `@shopify/flash-list` v2.
**Layout:** Fills the body area between header and input row. Horizontal padding `SPACING.md` (16pt). Vertical padding `SPACING.xl` (32pt) top inside scroll (creates breathing room from header), `SPACING.md` (16pt) bottom.
**Data model:** Flat array of `ChatMessage` records:
```ts
type ChatMessage =
  | { id: string; role: 'user'; text: string; createdAt: number }
  | { id: string; role: 'assistant'; text: string; chart?: ChartPayload; createdAt: number; isError?: boolean }
  | { id: string; role: 'assistant-typing'; createdAt: number }
```
**FlashList config:**
- `estimatedItemSize={88}` (typical bubble + spacing)
- `getItemType` returns `role` value (`'user' | 'assistant' | 'assistant-typing'`) for layout caching
- `keyExtractor` uses `item.id`
- `inverted={false}` — newest at bottom; auto-scroll-to-end on new message arrival
- `onContentSizeChange` callback scrolls to end (smooth) when a new message is appended
**Inter-bubble spacing:**
- Same author consecutive messages: `SPACING.sm` (8pt)
- Different author transition: `SPACING.md` (16pt)
**Auto-scroll-to-end:** triggers on `messages.length` increase. Uses `ref.scrollToEnd({ animated: true })`.
**Accessibility:** `accessibilityLabel="Chat messages"`. Each bubble is independently focusable (announced in order).

### ChatBubbleAssistant

**Purpose:** Single assistant-authored answer.
**Alignment:** Left (`alignSelf: 'flex-start'`).
**Max width:** 80% of list content width.
**Background:** `COLORS.surface` (`#FAF5F0`) — same as sheet body (intentional: assistant bubbles feel anchored to the sheet, not floating cards).
**Border:** 1pt `COLORS.textMuted` @ 20% — provides edge definition without harsh contrast.
**Corner radius:** `RADIUS.lg` (16pt) on top-left, top-right, bottom-right; **4pt** on bottom-left (tail-side).
**Internal padding:** `SPACING.md` (16pt) horizontal, `SPACING.md` (16pt) vertical.

**Content layout (top to bottom inside bubble):**
1. **Optional lead phrase** (used when answer includes a TL;DR sentence): `TYPE.editorialLead` (20pt EB Garamond 600) `COLORS.textPrimary`. Followed by `SPACING.sm` (8pt) gap.
2. **KPI number** (rendered when `chart.kpi` is present in the payload): `TYPE.displayM` (28pt Oswald 500) with `fontVariant: ['tabular-nums']`. Color: `COLORS.accent` for expense KPIs (negative or spend amounts), `COLORS.sage` for income/savings KPIs. Centered horizontally inside the bubble. Followed by `SPACING.sm` (8pt) gap.
3. **Editorial body**: `TYPE.editorialBody` (EB Garamond 400, 16pt) `COLORS.textPrimary`. Wraps freely. The full prose explanation goes here.
4. **Mini chart** (rendered when `chart` present): see `ChatMiniChart` below. `mt: SPACING.md` (16pt).

**Error variant (`isError === true`):**
- Single line: `TYPE.uiLabel` (14pt Manrope) `COLORS.error` — `"The assistant hit a snag. Please try again."` (`chat.error_assistant_inline`). Background `COLORS.error` @ 8% opacity, border 1pt `COLORS.error` @ 30%. No chart, no KPI.

**Timestamp on tap:** Same behavior as user bubble. Left-aligned below bubble. `accessibilityHint` mentions the gesture.

**Entrance animation:** Per §Animation Contract.

**Accessibility:** `accessibilityRole="text"`. `accessibilityLabel` composed dynamically:
- Normal: `"Assistant said: [text]. [If KPI:] Key number: [formatted KPI]. [If chart:] With a chart."`
- Error: `"Assistant error: [text]. Double-tap to retry."` — `accessibilityRole="button"` in this case.
- Long-form bodies use `accessibilityLiveRegion="polite"` on the bubble container so VoiceOver announces newly-arrived assistant content.

### ChatMiniChart

**Purpose:** In-bubble Skia chart embedded inside an assistant answer (CHAT-03).
**Library:** `@shopify/react-native-skia` only. NOT Victory Native (kept consistent with Phase 2 donut decision).
**Canvas size:**
- Sparkline: width = bubble inner width, height = 64pt
- Mini donut: 96×96pt, centered horizontally inside bubble
- Mini bar (compare): width = bubble inner width, height = 64pt
**Chart payload contract** (from `ai-query` Edge Function response — see 03-AI-SPEC.md):
```ts
type ChartPayload =
  | { kind: 'sparkline'; values: number[]; kpi?: number; unit?: 'EUR' }
  | { kind: 'donut'; slices: { label: string; value: number; color: 'accent' | 'sage' | 'accentSoft' | 'textMuted' }[]; kpi?: number; unit?: 'EUR' }
  | { kind: 'bar'; bars: { label: string; value: number }[]; kpi?: number; unit?: 'EUR' }
```
**Sparkline:** Single Skia `Path` line, 1.5pt stroke, `StrokeCap.Round`, `COLORS.accent`. No fill, no axis lines. X-axis evenly distributes values; Y-axis auto-scales (min 0, max = highest value). 
**Donut:** Pure Skia arcs (same arc math as Phase 2 donut). Stroke width 8pt (slightly thinner than Phase 2's 10pt because the canvas is smaller). Slice gap 2pt. Colors resolved from token names in the payload (never raw hex from the API — protects against the LLM emitting banned colors). Optional 2-line axis labels under donut: `TYPE.uiLabel` `COLORS.textMuted`, max 2 labels rendered (the two largest slices), `SPACING.xs` (4pt) gap.
**Bar:** Up to 5 vertical bars, equally spaced, width auto-calculated. Bar fill `COLORS.accent`. 1pt rounded top corners (2pt `RADIUS.sm`-equivalent). Labels `TYPE.uiLabel` `COLORS.textMuted` directly under each bar (truncated to 8 chars + ellipsis). Y-axis auto-scales.
**Color safety gate:** Before rendering, validate every `color` field in the payload is one of `'accent' | 'sage' | 'accentSoft' | 'textMuted'`. Unknown values fall back to `COLORS.textMuted`. The Edge Function MUST emit token names only — never raw hex. This is a UI-side guardrail layered on top of the AI-SPEC's response schema.
**Entrance animation:** Per §Animation Contract (Skia path stroke from 0→1 progress over 600ms).
**Accessibility:** The chart itself is announced via the parent bubble's `accessibilityLabel` (which includes "With a chart"). VoiceOver does not focus the chart independently — the textual prose above already conveys the data narratively. (Aligns with QUAL-03 — no dead-end focuses.)

### ChatErrorBanner

**Purpose:** Service-unavailable banner (ROADMAP success criterion #5).
**Position:** Inside the chat sheet, between the message list and the input row, anchored above the input row. Slides in/out per §Animation Contract.
**Height:** Auto (single line). Internal padding `SPACING.sm` (8pt) vertical, `SPACING.md` (16pt) horizontal.
**Background:** `COLORS.error` @ 10% opacity. 1pt top + bottom border `COLORS.error` @ 30%. No corner radius (full-width banner).
**Layout (left to right):**
- Error icon SVG (16pt) `COLORS.error`, `mr: SPACING.sm` (8pt)
- Body text: `TYPE.uiLabel` (14pt Manrope) `COLORS.error` — `"AI is unavailable. Tap to retry."` (`chat.error_unavailable`)
- Right-aligned: chevron-refresh SVG (16pt) `COLORS.error`
**Tap behavior:** Re-fires the last failed `ai-query` request. Haptic `Haptics.impactAsync('light')`. Banner stays visible while retry is in flight; dismisses on success; updates copy on second failure: `"Still unavailable. Check your connection."` (`chat.error_unavailable_retry`).
**Auto-dismiss:** None (stays visible until tapped or until user closes the chat sheet).
**When NOT shown:** Network is fine but the response is empty (no relevant data) — that case uses the inline empty-result variant inside an assistant bubble (`"I couldn't find data for that question. Try rephrasing?"`, `chat.empty_result`). The banner is reserved for Edge Function returning non-200 or no network.
**Accessibility:** `accessibilityRole="button"`, `accessibilityLabel="AI is unavailable. Tap to retry."`, `accessibilityLiveRegion="assertive"` (announces on appearance — this is a failure state user should hear immediately).

### PropagationToast

**Purpose:** Non-blocking confirmation that an AI propagation event updated similar transactions (CAT-04 user feedback).
**Trigger:** Fires after the user closes `RecategorizeBottomSheet` AND the `merchant_overrides` propagation pass updates ≥ 1 other transaction (silent if 0 other matches).
**Position:** Anchored to bottom of screen, above the tab bar by `SPACING.md` (16pt). Slides up from below per §Animation Contract.
**Width:** Screen width minus 2 × `SPACING.md` (32pt total inset).
**Height:** 56pt.
**Background:** `COLORS.surface`. `RADIUS.md` (12pt) all corners.
**Left edge:** 2pt vertical stripe `COLORS.success` (`#7A876A`) — subtle success cue.
**Shadow:** `SHADOWS.modal` (deeper than card; floats convincingly).
**Internal padding:** `SPACING.md` (16pt) horizontal, `SPACING.sm` (8pt) vertical.
**Layout (left to right):**
- Body text: `TYPE.uiLabel` (14pt Manrope) `COLORS.textPrimary`. Pluralized:
  - 1 other: `"Updated 1 similar transaction."` (`chat.propagation_one` — note: in `chat.*` namespace per i18n decision but conceptually a tx-list toast; alternative is `transactions.propagation_one` — see Copywriting table for final choice)
  - 2+: `"Updated {{count}} similar transactions."` (`chat.propagation_many`)
- Right-aligned: `"Undo"` action — `TYPE.uiLabel` semibold (Manrope 600) `COLORS.accent`. Tap target: 44×44pt via `hitSlop`.
**Tap behavior on toast body (NOT the Undo action):** No action — toast just acknowledges the event.
**Tap behavior on Undo:** Reverts the propagated category updates in DB (single transaction containing all rolled-back rows). Haptic `Haptics.notificationAsync('warning')`. Toast fades out 150ms. The originally-corrected row (the one the user explicitly swiped) is NOT reverted — only the auto-propagated rows.
**Auto-dismiss:** 4000ms after appearance, slides out per §Animation Contract.
**Stack behavior:** If a second propagation fires before the first toast dismisses, the first is replaced (not stacked). This avoids toast clutter.
**Haptic on appearance:** `Haptics.impactAsync('medium')`.
**Accessibility:** Root `accessibilityRole="alert"`, `accessibilityLiveRegion="polite"`, `accessibilityLabel="Updated [count] similar transactions"`. Undo: `accessibilityRole="button"`, `accessibilityLabel="Undo updates"`, `accessibilityHint="Reverts the auto-applied category changes"`.

## Privacy & Security Visual Contract (CHAT-04 enforcement at UI layer)

The UI MUST NOT display raw transaction descriptions inside chat bubbles or pass them to the Edge Function. Specifically:

- The chat input is **free-form user text only** — typed by the user. The UI never auto-fills it with transaction-description text from the DB.
- Mini chart payloads (see ChatMiniChart) accept only **token names** for colors (`'accent' | 'sage' | 'accentSoft' | 'textMuted'`). If the LLM emits raw hex, the UI falls back to `textMuted` (defense in depth — the AI-SPEC's response schema is the first guard; this is the second).
- Assistant prose may reference category names and aggregate numbers freely. It MUST NOT reference specific merchant descriptions from individual transactions. The Edge Function system prompt enforces this; if a response leaks a merchant string, that is an AI-SPEC issue, not a UI-SPEC issue — but the UI surfaces a `chat.error_assistant_inline` if a runtime content filter on the response detects a leak (post-MVP refinement; for Phase 3 we trust the system prompt).
- The AI-indicator dot exposes no transaction content — it is a single boolean visual.
- The propagation toast does NOT name the merchant pattern that was matched. It says `"Updated N similar transactions"` only. Generic by design.

This complements the AI-SPEC contract; the UI's job is to **never escalate** what the backend sends.
